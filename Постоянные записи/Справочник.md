### Что такое Справочник

Объект конфигурации *Справочник* предназначен для работы со списками данных.

Например:
1. Списки сотрудников
2. Списки товаров
3. Списки поставщиков

*Свойства* и *структура* этих списков описываются в объектах конфигурации Справочник, на основе которых платформа создает в базе данных таблицы для хранения информации из этих справочников.

*Справочник состоит из элементов.* Например, для справочника сотрудников элементом является сотрудник, для справочника товаров – товар и т. д.

В базе данных каждый элемент справочника представляет собой отдельную запись в основной таблице, хранящей информацию из этого справочника.

---
### Реквизиты Справочника

Для описания набора дополнительной информации, которую содержит каждый элемент справочника используются *[[Реквизит|реквизиты]]*.

*[[Реквизит|Реквизиты]]* справочника также, в свою очередь, являются [[Объект конфигурации|объектами конфигурации]].

У справочников существуют *стандартные реквизиты*, среди которых:
1. Код
2. Наименование

Также в том случае, если справочник иерархический, у каждого *элемента справочника* может стать доступен стандартный реквизит *Родитель*.

Также в том случае, если справочник подчинен другому объекту конфигурации, у каждого *элемента справочника* может стать доступен стандартный реквизит *Владелец*.

Таким образом, в базе данных справочник хранится в виде *таблицы*,
в строках которой расположены *элементы списка*, а каждому реквизиту (стандартному или созданному разработчиком) в этой таблице соответствует *отдельный столбец*. Соответственно, в ячейках этой таблицы хранится значение конкретного реквизита для конкретного элемента справочника (рис. 3.3).

### Табличные части в Справочнике

Кроме этого, каждый *элемент* справочника может содержать набор информации, которая *одинакова по структуре*, но *отличается по количеству* и *предназначена для разных элементов справочника*.

Для описания подобной информации могут быть использованы [[Табличная часть|табличные части]]объекта конфигурации *Справочник*, являющиеся подчиненными ему [[Объект конфигурации|объектами конфигурации]].

В этом случае в базе данных будут созданы дополнительные таблицы для хранения табличных частей, подчиненных конкретному элементу справочника.

В базе данных для справочника создаются несколько таблиц, эти таблицы связываются по уникальному полю (Ссылка), поля таблиц имеют определенные типы и т. д. Все это система делает сама. Нам лишь нужно добавить в объект конфигурации Справочник подчиненный ему объект [[Табличная часть|ТабличнаяЧасть]].

### Иерархические Справочники

Для удобства использования элементы справочника *могут быть сгруппированы*пользователем по какому-либо *принципу*.

*Иерархия* в Справочнике может быть двух типов:
1. *Иерархия групп и элементов* - когда мы создаем отдельные группы, в которые можем помещать элементы справочника.
	В этом случае элемент справочника, представляющий собой группу, будет являться *родителем* для всех элементов и групп, входящих в эту группу.
	Например, в справочнике *Бытовая техника* могут быть созданы следующие группы: *Холодильники*, *Телевизоры*, *Стиральные машины* и т.д.

2. *Иерархия элементов* - в этом случае в качестве родителя выступает не группа элементов справочника, а непосредственно один из его элементов.
	Например в случае справочника *подразделений* один элемент справочника, например *Московское подразделение*, будет *родителем* для всех *районных подразделений*.

### Формы справочника

В зависимости от того, какие действия мы хотим выполнять со справочником, нам требуется изображать справочник в «разном виде». Например, для того чтобы выбрать некоторый элемент справочника, удобнее представить справочник в виде списка, а для того чтобы изменить какой-то элемент справочника, удобнее представить все реквизиты этого элемента справочника в одной форме.

Набор *стандартных Форм* для объекта конфигурации [[Справочник]]:
1. *Форма элемента справочника* - тип: *Форма объекта*
2. *Форма группы справочника* - тип: *Форма группы
3. *Форма списка справочника* - тип: *Форма списка*
4. *Форма выбора справочника* - тип: *Форма выбора*
5. *Форма выбора группы справочника* - тип: *Форма выбора*

Подробнее про формы объекта конфигурации: [[Форма объекта конфигурации]].

### Справочник с предопределенными элементами

В ситуации, когда бизнес-процессы компании требуют *обязательного* наличия некоторых *стандартных элементов справочника*, без которых работа учета невозможна, существует механизм создания *предопределенных элементов справочника*.

*Предопределенный элемент справочника* - элемент *Справочника*, созданный на уровне конфигурации. Такие элементы создаются в базе данных автоматически.
